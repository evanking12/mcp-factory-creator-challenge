Overview
MCP Factory makes opaque compiled enterprise systems usable by AI agents.

It does that by recovering evidence, turning it into structured artifacts, and refusing to trust outputs that cannot be verified.

This demo is a static replay of real local runs. Live execution stays local because it requires user-owned model credentials, binary-analysis dependencies, and safe execution boundaries.
1 Start with the hard input.

A compiled/no-source enterprise artifact enters the pipeline. Source may exist for synthetic evaluation, but the product route cannot rely on it.

2 Recover evidence, not guesses.

The pipeline uses route authority, target analysis, semantic comparison, and durable artifacts to make recovered behavior reviewable.

3 Gate what agents can trust.

Outputs become product-green, comparable, diagnostic, frontier, or blocked based on evidence, not model confidence.

How The Test Suite Builds Confidence

easy to hard

The suite is not a random benchmark list. Each step is authored to break a different part of the product route before the next step is trusted.

Easy01 Can the route work at all?

Proves the compiled/no-source route can admit the artifact, recover readable evidence, and reach a basic semantic checkpoint.

Medium01 Can attribution survive more targets?

Adds shared-code and multi-target pressure so evidence cannot be blended across recovered callable surfaces.

Medium02 Can comparability stay repeatable?

Acts as the current product-route anchor for route authority, repeatable semantic proof, and safe-to-compare behavior.

Hard01 Can enterprise decisions survive?

Stresses branch trust, rejected-object evidence, replay trust, closeout, and claim ceilings under harder fixture behavior.

Hard02 / Hard03 Where does the product still fail closed?

Shows frontier and blocker cases: stale authority, object-bound decisions, unsupported-target quarantine, and unsafe-path handling.

Batch B Can this scale beyond one curated case?

Package-scale pressure checks whether the same route and evidence contracts survive across a broader compiled fixture batch.

Core Engineering Principles

engineering philosophy

MCP Factory is built around evidence boundaries: agents can help operate the system, but artifacts and gates decide what can be trusted.

01 Compiled/no-source first

Prove behavior from the artifact the product actually receives. Synthetic source can help evaluate the fixture, but it cannot become a shortcut in the product route.

02 Artifact-backed claims

No durable evidence, no stronger claim. Route summaries, stage artifacts, compare reports, traces, and closeout records are what make a replay reviewable.

03 Fail closed

Missing route, semantic, artifact, or closeout proof downgrades the run to diagnostic, frontier, or blocked instead of letting an LLM explanation overrule the evidence.

04 Smallest proof surface

Use targeted checkpoints and stage-local evidence before spending expensive full reruns. The pipeline should localize the first untrusted boundary.

05 Suites as breakpoints

Easy, medium, and hard cases are authored to attack different seams: route entry, attribution, comparability, branch trust, stale authority, and unsafe-path handling.

06 Honest claim ceilings

Replay-backed, product-green, comparable, frontier, diagnostic, and blocked mean different things. The UI preserves those distinctions so the demo does not overclaim.
