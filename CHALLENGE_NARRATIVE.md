# Creator Challenge Narrative

## Demo Claim

MCP Factory turns compiled/no-source enterprise inputs into staged,
reviewable, agent-usable evidence. This page is a static replay of real run
artifacts, not a hosted execution environment.

## Difficulty Ladder

- `easy01`: happy-path product proof for a controlled compiled fixture.
- `medium01`: broader shared-code pressure while preserving route authority.
- `medium02`: current medium product-route anchor for comparability and
  semantic alignment.
- `hard01`: enterprise-survivability pressure with branch and semantic
  continuity requirements.
- `hard02`: stale-attestation survivability and object-bound authority
  pressure. Current public demo status is a frontier signal, not a replay run:
  comparable private truth exists, but no sanitized public replay root is
  bundled here.
- `hard03`: unsupported-target quarantine / unsafe-path denial pressure.
  Current status is an active blocker frontier around reservation-stage analyze
  authority and reopen-gate proof, not public product proof.
- `Batch B`: package-scale intake pressure; useful for breadth and blocker
  harvest, not product-green proof.

## Artifact Evidence

For replay-backed runs, the demo shows the stage artifacts the pipeline
consumed, emitted, and used as proof. Expected behavior is represented through
the stage contracts and sanitized sidecars rather than a separate
observed-versus-expected widget, which could make the demo look like a source
comparison instead of a compiled/no-source replay.

## Engineering Story

The public demo avoids a live backend because live execution requires local
credentials, a binary-analysis bridge, Ghidra, and trusted machine-local
services. Instead, it shows the evidence a real run produced: stages, artifacts,
comparability, blockers, and the next bounded action.

OpenTelemetry-shaped JSON is used as a planned diagnostic trace spine. Current
metrics are derived from replay artifacts, estimates, or explicit unavailable
placeholders. They help explain what would be useful to observe next, but they
do not replace semantic proof, `safe_to_compare`, or product-green evidence.

## GM Dispatch Accountability

- UI replay auditor: dispatched read-only to compare the static demo against
  the `/simple` cockpit style.
- Telemetry contract auditor: dispatched read-only to check the replay schema,
  priority metrics, identity labels, and sanitization risks.
- Final verification reviewer: required before publication to confirm the
  static page does not overclaim live execution or leak local-only details.

## Public Boundary

No public repo mutation or GitHub Pages deployment is allowed until the private
static demo passes local static serving, schema validation, and leak scanning.
The public showcase repo should contain only the static demo, sanitized replay
artifacts, and narrative docs: no keys, compiled inputs, raw sessions, or
private implementation internals.
